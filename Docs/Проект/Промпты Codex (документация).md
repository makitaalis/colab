# Промпты Codex (документация)

Набор промптов для модульного ведения документации проекта.

## 1) Аудит структуры документации

```text
Проведи аудит архитектуры документации проекта.
Сделай:
1) карту файлов Docs/Проект;
2) проверку соответствия слоям L0-L4;
3) список дублирования/пробелов;
4) конкретный план правок (без изменения канона).
После правок обнови Docs/Проект/INDEX.md.
```

## 2) Обновление docs после этапа работ

```text
Обнови документацию после выполненного этапа <phase>.
Требования:
1) зафиксировать изменения в профильном документе;
2) обновить операции (команды проверки/выкатки);
3) синхронизировать skills/prompts при необходимости;
4) проверить, что ссылки есть в Docs/Проект/INDEX.md;
5) в конце явно добавить: что будет сделано следующим этапом и зачем.
```

## 3) Создание нового модульного документа

```text
Создай новый модульный документ для темы <topic> по шаблону:
Docs/Проект/Шаблон документа модуля.md

Требования:
1) одна зона ответственности;
2) разделы: Назначение, Контракт, Операции, Типовые проблемы;
3) добавить ссылку в Docs/Проект/INDEX.md;
4) не дублировать канон door_id/network/time.
```

## 4) Синхронизация канона и рабочих документов

```text
Сверь канон и рабочие документы.
Проверь:
1) соответствие door_id ↔ hostname ↔ IP;
2) базовую сеть и central_id/system_id правила;
3) отсутствие старых/конфликтующих формулировок.
При расхождениях обнови документы и перечисли, что изменено.
```

## 5) Масштабирование документации под 100-200 систем

```text
Подготовь документацию к масштабу 100-200 систем.
Сделай:
1) обнови runbook массового ввода system_id;
2) приведи правила WG/API-key/commissioning к единому алгоритму;
3) проверь ссылки на fleet/registry.csv и fleet/out/<SYSTEM_ID>/commands.md;
4) добавь checklist "готово к вводу нового транспорта".
```
