# Домен Старт

## Назначение

Стартовые страницы для быстрого входа в систему и onboarding нового транспорта.

## Route scope

- `/admin` (`overview`)
- `/admin/commission` (`commission`)

## API scope

- `/api/admin/fleet/monitor`
- `/api/admin/fleet/centrals`
- `/api/admin/wg/peers`
- `/api/admin/fleet/incidents`

## UX-контракт домена

1. Видимая summary-строка контекста.
2. Быстрые переходы в соседние домены.
3. Empty-state с понятным следующим шагом.
4. Перегруженные операционные блоки выносятся во secondary (`<details>`), primary-экран отвечает на 1 вопрос.

## Контент-раскладка (commission)

Страница `/admin/commission`:

1. Primary:
- “Комісія: статус і наступні дії” (backend + WG + incidents totals + next actions).
- “Команди (копіпаст)”.
2. Secondary (persistent):
- commissioning чеклист,
- локальные `runs`,
- локальные `profiles`.

Persistence key:
- `passengers_admin_commission_secondary_details_v1`.

## Ограничения

- без deep triage таблиц (это домен `Флот`);
- без массовых действий над алертами (это домен `Флот`/`Контроль і KPI`).

## Definition of Done

1. `/admin` и `/admin/commission` отвечают `200`.
2. Есть `copyLink`, `filterSummary`, `flowStep`.
3. Подключение транспорта проходит по чеклисту без выхода за пределы домена.
